## [RFC 10008: The new HTTP Query Method](https://www.rfc-editor.org/info/rfc10008/) ⭐️ 8.0/10

RFC 10008 introduces the new HTTP QUERY method, which allows for the inclusion of request bodies in idempotent requests. This method aims to standardize how such requests are handled in web communications. The introduction of the QUERY method is significant as it provides a standardized way to perform complex queries without the limitations of the GET method. This change could impact API design and improve the handling of requests in web applications. The QUERY method is designed to be safe and idempotent, meaning it can be repeated without side effects. It also allows for more complex query structures by including them in the request body rather than the URL.

hackernews · schappim · Jun 17, 10:51

**Background**: HTTP defines various request methods, including GET and POST, for different purposes in web communications. The GET method traditionally does not support request bodies, which can limit its use in certain scenarios. The QUERY method aims to fill this gap by allowing request bodies while maintaining idempotency.
